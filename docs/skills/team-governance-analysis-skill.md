# Team & Governance Analysis Skill

## Purpose
Analyze team credibility, governance structure, and decision-making processes.

## Inputs
- Token contract address (required)
- Network/chain (required)
- Official project documentation (recommended)
- Social media presence (X, Discord, Telegram, Medium)

## Loop 1 — Team attribution
Gather team member information:

**Core Team Members**
- Name (pseudonym/known)
- Role (Founder, Developer, Designer, PM)
- LinkedIn/GitHub profile
- Time commitment (full-time/part-time/advisory)
- Track record (previous projects, exit achievements)
- Reputation (on-chain/social)

**Advisors & Partners**
- Investment portfolio (other protocols)
- Domain expertise (DeFi, NFT, Gaming)
- Network connections (exchange, VC, incubator)
- Advisory roles (multiple projects)

**Investors & Backers**
- VC/angel investors
- Strategic backers (exchanges, protocols)
- Grant programs (EC, DAOs)
- Institutional backers
- Investment thesis alignment

**Team Structure**
- Legal entity (if incorporated)
- Geographic distribution
- Compensation method (token vesting, salary)
- Transparency level (transparent/obscure)

## Loop 2 — Social proof & communication
Analyze project transparency and community engagement:

**Social Metrics**
- Discord/Telegram members
- X (Twitter) followers
- Reddit karma
- GitHub stars/forks
- Blog/Medium posts

**Communication Quality**
- Transparency in decision-making
- Community engagement (response times)
- Regular updates (weekly/monthly/quarterly)
- Developer documentation quality

**Transparency Assessment**
- On-chain voting record
- Budget transparency
- Code repository visibility
- Bug bounty program presence

## Loop 3 — Governance structure analysis
Analyze decision-making mechanisms:

**Protocol Governance Types**
- Token holder voting (ERC-20)
- Multi-sig governance (DAO)
- Community vote platforms (Snapshot, guild)
- On-chain vs. off-chain voting
- Voting power distribution (token-weighted, quadratic, weight-of-knowledge)

**Governance Parameters**
- Voting threshold (required majority)
- Proposal creation cost
- Voting duration
- Execution delay
- Upgrade authority
- Emergency control

**Governance Participation**
- Active voter participation rate
- Proposal creation frequency
- Emergency proposal success rate
- Governance quality (well-reasoned vs. rushed)

## Loop 4 — Key decision-making analysis
Analyze major decisions and their quality:

**Protocol Changes**
- Upgrade frequency
- Parameter changes
- Contract modifications
- Risk assessment of upgrades

**Community Decisions**
- Treasury management
- Partnerships
- Integrations
- Strategic pivots
- Competitive positioning

**Quality Assessment**
- Decision speed vs. quality
- Risk considerations
- Stakeholder alignment
- Transparency of process

## Loop 5 — Track record analysis
Analyze team's previous successful achievements:

**Successful Projects**
- Previously completed projects
- Revenue streams/launch success
- Market adoption metrics
- Community growth

**Lessons Learned**
- Project failures and recovery
- Strategic pivots and rationale
- Competitive response
- User feedback incorporation

## Data Sources
- LinkedIn profiles
- GitHub repositories
- Company websites/investor pages
- Annual reports/Whitepapers
- Social media platforms
- Community forums
- On-chain governance forums (Snapshot, Aragon)
